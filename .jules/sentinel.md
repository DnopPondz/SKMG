## 2024-03-24 - [CRITICAL] Prevent NoSQL Injection via Unvalidated Request Payload in Next.js App Router
**Vulnerability:** Passing unvalidated `req.json()` payloads directly into Mongoose queries (e.g., `findOne` or `create`) in Next.js API Routes introduces critical NoSQL injection and mass assignment risks.
**Learning:** Next.js App Router API endpoints directly expose JSON body inputs which, if not typed and validated properly, can pass malicious query operators (like `$ne`, `$gt`) directly to MongoDB, bypassing authentication or manipulating logic. Additionally, the standard practice in Next.js of destructuring `await req.json()` doesn't offer runtime type safety.
**Prevention:** Always use Zod's `safeParse()` to validate incoming JSON payloads. Do not expose `ZodError` details directly in `NextResponse.json()` as it leaks internal validation rules and causes TypeScript build failures. Never return `error.message` or stack traces in catch blocks; return a generic 500 "Internal server error" message.
