## 2026-05-08 - [Sentinel] Input Validation and Secure Error Handling in API Route\n**Vulnerability:** The API route for adding products passed raw request data to the database without validation, creating a mass assignment risk. Additionally, the generic catch block was exposing error details.\n**Learning:** In Next.js applications dealing with MongoDB, directly writing `req.json()` into `Model.create()` is insecure. Using Zod allows ensuring that only safe data is persisted, and catch blocks must omit the error to avoid leaking implementation details.\n**Prevention:** Use Zod validation schemas for all incoming data and avoid exposing `error.message` directly from generic catch blocks in the App Router API handlers.
