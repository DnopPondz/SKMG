## 2024-05-01 - Input Validation and Generic Errors Prevent NoSQL Injection & Information Leakage
**Vulnerability:** The API endpoint `src/app/api/stock/route.ts` processed unvalidated `req.json()` payloads directly into Mongoose queries and arithmetic operations. If `amount` was a negative number, it bypassed the business logic check (`newQuantity < 0`) allowing malicious stock additions via the `OUT` type. In addition, the error handler leaked raw error messages to the client.
**Learning:** In Next.js API routes with MongoDB, failing to parse untyped `req.json()` opens endpoints to critical logic bypasses, negative value manipulation, and NoSQL injection.
**Prevention:** Always use a schema validation library like `zod` to enforce data types, positive numbers (`z.number().positive()`), and structural integrity before using the data in queries or business logic. Fail securely by replacing specific `error.message` with a generic "Internal server error" string to avoid data leaks.
