## 2024-05-18 - Passing unvalidated payload from Next.js req.json() to Mongoose models
**Vulnerability:** The application was vulnerable to NoSQL Injection/Mass Assignment. Endpoints were directly passing the raw JSON body (`req.json()`) into Mongoose methods such as `Product.findOne({ sku })` and `Product.create({ ...data })`. Since JSON objects can represent NoSQL query operators (like `{"$ne": null}`), this could allow attackers to bypass checks, retrieve unexpected records, or mass assign properties on product creation.
**Learning:** In Next.js App Router API routes, data returned from `req.json()` must be considered completely untrusted. Mongoose does not sanitize operator objects if they match schema keys without strictly enforcing types.
**Prevention:** Always validate and sanitize `req.json()` payloads using a schema validation library like `zod` to enforce strict primitive types (e.g., `z.string()`, `z.number()`) and strip out unrecognized keys before interacting with the database.
