## 2024-05-31 - [Mass Assignment and NoSQL Injection via Next.js App Router req.json()]
**Vulnerability:** Passing unvalidated `req.json()` payloads directly into Mongoose queries (e.g., `findOne`) or create operations in Next.js App Router introduces mass assignment and NoSQL injection risks.
**Learning:** Developers often pass the raw payload from Next.js App router `req.json()` into mongoose operations because it looks like a regular javascript object. However, this allows attackers to send unexpected properties (like `$gt` or fields they shouldn't modify, or bypassing schema validation in unexpected ways before it reaches mongoose) leading to data manipulation and potentially unauthorized access. Also, error messages from Mongoose were being leaked directly to the client which can expose implementation details.
**Prevention:** Use Zod `safeParse()` to strictly validate all incoming data from `req.json()` before use. Check `result.success` and always use the parsed `data` object instead of the raw input. Do not leak validation errors directly by using `result.error.errors` or Mongoose exceptions in the catch block to the client, return a generic error message (e.g., `{ message: "Invalid input data" }` or `{ message: "Internal server error" }`) instead.
