## 2024-05-17 - Prevent NoSQL Injection in Mongoose queries using Zod Validation
**Vulnerability:** In `src/app/api/stock/route.ts`, an untyped and unvalidated `sku` field from `req.json()` was passed directly to a Mongoose query (`Product.findOne({ sku })`). An attacker could supply an object (like `{"$ne": null}`) instead of a string, bypassing intended string lookups and causing a NoSQL injection.
**Learning:** In Next.js App Router API routes, data extracted via `req.json()` must not be implicitly trusted when interacting with MongoDB/Mongoose. Unvalidated inputs can directly exploit Mongoose's query operators.
**Prevention:** Always define a strict `zod` schema for API request bodies and validate the input before executing any database queries. Ensure fields used in database lookups are explicitly validated as strings (or numbers/booleans as expected). In addition, when updating error handling for validation, ensure the generic error payload does not expose stack traces or raw validation errors to avoid leaking internals.
