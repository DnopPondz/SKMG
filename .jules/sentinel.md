## 2026-05-05 - [CRITICAL] Prevent NoSQL Injection & Data Leakage in API Routes
**Vulnerability:** The API routes (`src/app/api/products/add/route.ts` and `src/app/api/stock/route.ts`) were vulnerable to NoSQL injection and Mass Assignment due to passing raw, unvalidated `req.json()` directly into Mongoose queries and object creation operations. Additionally, the applications leaked stack traces/sensitive details by passing `error.message` directly back to users in `catch` blocks.
**Learning:** Using the App Router in Next.js, API request bodies should not be trusted and must be validated before reaching MongoDB queries. Exposing error details via generic `catch (error: any)` structures facilitates reconnaissance attacks.
**Prevention:** Implement strict Zod schema validation for all incoming API data to explicitly whitelist expected fields and drop malformed data. Use generic error messages (e.g., "Internal server error") in catch blocks instead of the native error messages.
