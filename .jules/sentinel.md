## 2024-05-24 - [CRITICAL] Prevent NoSQL Injection in Mongoose queries using unvalidated JSON payloads
**Vulnerability:** Passing unvalidated `req.json()` payloads directly into Mongoose queries (e.g., `findOne({ sku })` where `sku` is destructured from `req.json()`) in Next.js App Router introduces NoSQL injection risks. An attacker could send query operator objects like `{"sku": {"$ne": null}}` to bypass checks or access unauthorized records.
**Learning:** `req.json()` parses any JSON structure, including objects used as NoSQL operators. Destructuring directly into a Mongoose query without validating that the value is a primitive (like a string) allows these objects through.
**Prevention:** Always use a validation library like `zod` to enforce string types and validate all API inputs before using them in database queries. Additionally, return generic error messages (e.g., "Invalid input data") for validation failures to prevent leaking internal details or causing TypeScript build failures with `ZodError`.
